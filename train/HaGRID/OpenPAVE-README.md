# OpenPAVE-README — why this data is on disk

AI training data for hand gesture recognition: a pruned image dataset
(YOLO layout) generated by `train/hagrid_yolo.py prepare` from the
[HaGRID](https://github.com/hukenovs/hagrid) sample archive
(`cj-mills/hagrid-sample-30k-384p` on HuggingFace, cached once in
`~/.cache/huggingface` alongside the OpenPAVE VLM checkpoints).

YOLO training needs plain image + label files on a local path — it cannot
stream from a zip in the HF cache — so the selected slice is materialised
here. It is disposable: delete `images/`, `labels/`, `data.yaml` and
`dataset_stats.json` at any time and re-run `./train/HaGRID.sh prepare`.

## Details

- Classes: N gesture classes (default `stop, fist, like, point_up,
  point_right, point_down, point_left` — the STOP/HOME/TROT/UP/RIGHT/DOWN/
  LEFT intents in `pave_ui/perception.py`) + `no_gesture` as a distractor so
  stray hands don't fire intents. HaGRID has no directional point classes:
  `point_up` is HaGRID's `one` (index finger up) as-is, and the other three
  directions are synthesized by rotating disjoint slices of `one` frames
  (whole image + bboxes, 90°/180°/270°). Rotated frames look odd (sideways
  people) — good enough to prove the direction; replace with real captured
  point data when accuracy matters.
- Size: capped per class (default 300 images, ~15% val split) — deliberately
  small; enough to prove the YOLO-nano-on-Mali direction, not a production
  training set.
- Layout: `images/{train,val}/`, `labels/{train,val}/` (YOLO txt,
  class cx cy w h normalised), `data.yaml`, `dataset_stats.json`.
- License: HaGRID images are distributed under CC BY-SA 4.0 — fine for
  training/evaluation here; attribution required if redistributed.
